## FAZ 3: Prompt Variable Sistemi (3-4 gün)
> Weavy'nin en güçlü özelliği — template değişkenleri

### 3.1 Variable-Aware Prompt Node
**Güncelleme:** Mevcut Script node → Prompt Node olarak genişlet
- Metin içinde `{variable_name}` syntax'ı
- Renkli highlight (turuncu arka plan)
- "+ Add variable" butonu
- Display modu: Value (çözümlenmiş) / Source (template)
- Sol port: Variable input'ları (dinamik, bağlanan variable sayısı kadar)

### 3.2 Datatype Node'ları
**Yeni node'lar:**
- **Number** — sayısal değer girişi
- **Text** — serbest metin
- **Toggle** — boolean (true/false)
- **List Selector** — dropdown listeden seçim (Weavy'deki gibi: Shot Size, Camera Angle, Lens...)
- **Seed** — rastgele seed değeri

### 3.3 Variable Çözümleme
**Güncelleme:** `execution-engine.ts`
- Prompt node çalıştırılırken upstream variable node'larının değerleri alınır
- `{variable_name}` → gerçek değerle replace edilir
- Çözümlenmiş metin downstream node'lara aktarılır

### Dosya değişiklikleri:
| Dosya | İşlem |
|-------|-------|
| `components/workspace/nodes/PromptNode.tsx` | YENİ — variable-aware prompt |
| `components/workspace/nodes/DatatypeNodes.tsx` | YENİ — Number, Text, Toggle, ListSelector, Seed |
| `lib/stores/workspace-store.ts` | GÜNCELLE — PromptConfig, DatatypeConfigs |

## FAZ 7: Yeni AI Provider Entegrasyonları (4-5 gün)
> fal.ai dışında doğrudan API entegrasyonları

### 7.1 Öncelikli Provider'lar
| Provider | Model Sayısı | Neden Öncelikli |
|----------|-------------|-----------------|
| Google (doğrudan) | Imagen 4, Veo 3.1, Gemini 3 | En geniş kapsam |
| Runway | Gen-4, Gen-4.5, Aleph | Premium video |
| Replicate | Community modelleri | Esneklik |

### 7.2 API Gateway Pattern
**Yeni dosya:** `lib/ai/providers/` klasörü
```
lib/ai/providers/
  ├── base-adapter.ts      — interface tanımı
  ├── fal-adapter.ts       — mevcut fal.ai (refactor)
  ├── google-adapter.ts    — doğrudan Google API
  ├── runway-adapter.ts    — Runway API
  ├── replicate-adapter.ts — Replicate API
  └── router.ts            — node_type → adapter mapping
```

### 7.3 Provider Adapter Interface
```typescript
interface AIProviderAdapter {
  name: string
  run(input: ProviderInput): Promise<ProviderOutput>
  estimateCost(input: ProviderInput): number
  checkAvailability(): Promise<boolean>
}
```
